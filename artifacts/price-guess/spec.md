# 물가 맞히기 게임 — spec

## 개요

농수산물의 실제 시세를 소재로 한 가격 맞히기 게임. 사용자는 한 게임에서 10개 품목의 가격대를 객관식으로 고른다. 오답이어도 게임은 끝까지 진행되며, 10문제를 푼 뒤 맞힌 개수(0~10)를 점수로 온라인 랭킹에 등록할 수 있다.

## 범위

### 포함

- 농수산물 품목 한 개를 제시하고, 가격대 4개 보기 중 하나를 고르는 객관식 라운드
- 정답/오답과 무관하게 다음 품목으로 진행하며, 진행도(현재 문제/총 10)와 맞힌 개수를 표시
- 매 문제마다 정답 공개 후 실제 가격 + 간단 설명(품목·단위·기준일·시장 구분 등) 표시
- 10문제를 모두 푼 뒤 최종 점수(맞힌 개수 / 10) 표시
- 닉네임을 입력해 점수를 온라인 랭킹에 등록하고, 전체 순위(상위권 + 내 순위)를 확인
- 가격 데이터를 불러올 수 없을 때의 안내(직전 캐시로 진행 / 재시도)

### 제외

- **사용자 계정/로그인** — 랭킹 등록은 닉네임만으로 한다. 영속적 프로필·내 기록 히스토리는 범위 밖. (게스트 진입 마찰을 없애기 위함)
- **농수산물 외 품목(생필품 등)** — MVP는 농수산물만. 데이터 커버리지가 가장 안정적이라 우선한다.
- **객관식 외 입력 방식(가격 직접 입력, Higher/Lower)** — 채점이 단순하고 진입이 쉬운 객관식으로 한정.
- **멀티플레이/실시간 대전** — 단일 플레이어. 경쟁은 비동기 랭킹으로만.
- **난이도 선택, 카테고리 필터** — MVP에서는 단일 난이도. 추후 확장 여지.

## 시나리오

### 1. 게임 시작

- **Given** — 시작 화면에 "시작" 버튼과 (있다면) 내 최고 기록이 표시된 상태
- **When** — "시작" 버튼을 클릭한다
- **Then** — 첫 번째 농수산물 품목명과 단위가 제시되고, 서로 다른 가격대 보기 4개, 진행도("문제 1 / 10")와 맞힌 개수(0)가 표시된다

성공 기준:
- [ ] "시작" 클릭 → 품목명·단위 + 가격대 보기 4개가 화면에 나타난다
- [ ] 보기 4개는 서로 다른 가격대이며, 그중 정확히 1개가 실제 가격이 속한 가격대다
- [ ] 화면에 진행도 "문제 1 / 10"과 "맞힘 0"(또는 동등한 표시)이 보인다

### 2. 정답 선택 → 다음 문제

- **Given** — 품목과 가격대 보기 4개가 표시된 문제 화면
- **When** — 실제 가격이 속한 가격대 보기를 선택한다
- **Then** — "정답" 표시와 함께 실제 가격·기준일·시장 구분이 담긴 설명이 나타나고, 맞힌 개수가 1 증가하며, "다음 문제" 진행 시 새 품목이 제시된다

성공 기준:
- [ ] 정답 보기 선택 → "정답" 표시 + "○○ <단위> N원 (○○시장, M월 D일 기준)" 형태의 실제 가격·설명이 보인다
- [ ] 정답 후 맞힌 개수가 직전보다 1 증가한 값으로 표시된다
- [ ] 다음 문제로 진행하면 새 문제가 표시되고 진행도가 1 증가한다

### 3. 오답 선택 → 게임은 계속

- **Given** — 품목과 가격대 보기 4개가 표시된 문제 화면 (마지막 문제 아님)
- **When** — 실제 가격이 속하지 않은 가격대 보기를 선택한다
- **Then** — "오답" 표시와 정답 보기·실제 가격이 나타나고, 맞힌 개수는 늘지 않으며, "다음 문제"로 게임이 계속된다

성공 기준:
- [ ] 오답 보기 선택 → "오답" 표시 + 실제 가격·정답 보기가 보인다
- [ ] 오답 후에도 맞힌 개수는 변하지 않고 "다음 문제" 버튼으로 계속 진행된다
- [ ] 공개(정답/오답) 후에는 해당 문제의 보기를 더 선택할 수 없다(비활성)
- [ ] 10번째 문제 공개 시에는 "다음 문제" 대신 "결과 보기"가 표시된다

### 4. 결과 → 랭킹 등록

- **Given** — 최종 점수(맞힌 개수 / 10)가 표시된 결과 화면, 닉네임 입력란과 "등록" 버튼
- **When** — 닉네임을 입력하고 "등록" 버튼을 클릭한다
- **Then** — 해당 점수가 랭킹에 반영되고, 랭킹 목록에서 방금 등록한 항목(닉네임·점수)이 강조 표시된다

성공 기준:
- [ ] 닉네임 "홍길동" 입력 후 "등록" → 랭킹 목록에 "홍길동 N" 항목이 나타나고 강조된다
- [ ] 닉네임이 비어 있는 상태로 "등록" 클릭 → "닉네임을 입력하세요" 안내가 표시되고 등록되지 않는다
- [ ] 등록 후에는 같은 결과를 중복 등록할 수 없다(등록 버튼 비활성 또는 "등록됨" 표시)

### 5. 랭킹 보기

- **Given** — 점수가 등록된 랭킹 목록
- **When** — 랭킹 화면을 본다
- **Then** — 상위권 항목들이 점수 내림차순으로 표시되고, 내가 방금 등록한 기록이 식별 가능하게 표시된다

성공 기준:
- [ ] 랭킹 목록이 점수 내림차순(높은 점수가 위)으로 정렬되어 표시된다
- [ ] 점수 50과 30을 등록하면 50이 30보다 위에 표시된다
- [ ] 내가 등록한 항목이 목록에서 강조(하이라이트/표식)되어 보인다

### 6. 가격 데이터 캐시로 진행 (API 장애, 캐시 있음)

- **Given** — 실시간 가격 조회는 실패하지만 직전에 성공적으로 받아둔 가격 데이터가 있는 상태
- **When** — 게임을 시작한다
- **Then** — 캐시된 가격을 기준으로 정상 진행되며, 가격 설명에 데이터 기준일이 명시된다

성공 기준:
- [ ] 실시간 조회 실패 + 캐시 존재 → 게임이 중단 없이 시작되고 문제가 표시된다
- [ ] 정답 공개 시 표시되는 기준일이 캐시 데이터의 날짜로 나타난다

### 7. 가격 데이터 없음 안내 (API 장애, 캐시 없음)

- **Given** — 실시간 가격 조회가 실패하고 사용할 캐시도 없는 상태
- **When** — 게임을 시작하려 한다
- **Then** — "가격 정보를 불러올 수 없어요" 안내와 "다시 시도" 버튼이 표시된다

성공 기준:
- [ ] 조회 실패 + 캐시 없음 → "가격 정보를 불러올 수 없어요" 메시지와 "다시 시도" 버튼이 보인다
- [ ] "다시 시도" 클릭 후 조회가 성공하면 게임이 시작된다

## 불변 규칙

- **데이터 정합성** — 모든 문제에서 가격대 보기 4개는 서로 겹치지 않고, 그중 정확히 1개에만 실제 가격이 속한다.
- **출처 투명성** — 실제 가격을 공개할 때마다 가격의 기준일과 시장/단위 정보를 함께 표시한다. 실데이터(KAMIS)가 아니라 AI(LLM) 추정 시세로 폴백한 경우 "AI 추정" 배지로 명시해, 사용자가 데이터 성격을 구분할 수 있어야 한다.
- **랭킹 일관성** — 랭킹에 표시되는 점수는 사용자가 실제로 달성한 연속 정답 수와 항상 일치하며, 정렬은 항상 점수 내림차순이다.

## 의존성

- 농수산물 실시간 가격을 제공하는 공공데이터 API (예: KAMIS 농수산물 가격정보) — API 키 발급 및 호출이 게임 시작 전 가능해야 한다.
- (폴백) KAMIS 키·캐시가 모두 없을 때 "AI 추정" 시세를 생성하는 LLM API (Gemini) — 선택적. 키가 있으면 게임이 중단 없이 진행된다.
- 랭킹 점수를 저장·조회할 수 있는 영속 저장소(서버 측).

## 미결정 항목

- **오답 보기 가격대 생성 규칙** — 정답 외 3개 가격대를 실제 가격으로부터 얼마나 떨어뜨려 생성할지(난이도 직결). 보기가 너무 쉽거나 어렵지 않도록 추후 plan 단계에서 규칙을 정한다.
- **닉네임 정책** — 중복 닉네임 허용 여부, 욕설/부적절 표현 필터링 여부.
- **랭킹 표시 개수** — 상위 몇 명까지 보여줄지(예: Top 10/Top 50).
- **품목 출제 풀과 중복 방지** — 한 게임 내 같은 품목 재출제 허용 여부.
